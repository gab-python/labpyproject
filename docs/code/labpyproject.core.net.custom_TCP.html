labpyproject.core.net.custom_TCP

Prototype de framework client/serveur visant rsoudre les questions souleves dans larticle : https://docs.python.org/3/howto/sockets.html

Protocole(s) support(s) : TCP

Principes :

  • toute requte est dcoupe (au besoin) en blocs de taille CustomRequestHelper.BUFFERSIZE

  • une couche de protocole identifie chaque bloc avec :

    • un prefix de bloc : CustomRequestHelper.BLOC_PREFIX (<#bp#>)

    • un entte : |n bloc/nombre de blocs|uid|nombre de caractres du message|

    • la portion de message du bloc

    • un suffixe de bloc : CustomRequestHelper.BLOC_SUFFIX (<#bs#>)

  • des codes (et arguments) de commande permettent de grer : les accuss de rception (permettant de garantir la rception dun message), lidentification unique dun client (uid), la communication de son adresse de rception (pour garantir le canal serveur vers client, cf limites de ThreadedTCPServer(socketserver.ThreadingMixIn, socketserver.TCPServer))

Exemple

Blocs binaires changs lors de la squence de connection.

1- Envoi dune demande duid par le client :

b<#bp#>|1/1|None|18|[cmd:ASK_FOR_UID|]<#bs#>

2- Rception du bloc par le Serveur, qui retourne lidentifiant uid0 au client :

b<#bp#>|1/1|gen_svr_id|4|uid0<#bs#>

3- Rception du bloc par le client, qui retourne ses infos de connection :

b<#bp#>|1/1|uid0|56|[cmd:SET_CLIENT_READ_INFOS|host=192.168.0.7&port=11548]<#bs#>

4- Rception du bloc par le serveur qui retourne un accus de rception :

b<#bp#>|1/1|gen_svr_id|26|[cmd:CONFIRM_RECEPTION|]56<#bs#>

5- Rception de la confirmation par le client avec le bon nombre de caractres (56).

La communication bilatrale est alors tablie.

Autres fonctionnalits :

  • possibilit de dconnecter / reconnecter les composants chaud.

  • dtection des dconnections sauvages distantes pour limiter le nombre de reconnection et renvois.

  • informe lapplication associe (BUSINESSComp) de ltat des diffrentes connections et des erreurs denvoi.

faire

Evolutions souhaitables:

  • supporter plusieurs codes, regrouper codes et msguid en entte

  • parallliser les envois de requtes ct serveur (voir mthode send du serveur frontal)

  • dcliner pour les trois autres protocoles supports par socketserver?

Application concrte

Dans le jeu labpyrinthe :

AppManager (voir labpyproject.apps.labpyrinthe.app.application, dans la mthode prive _init_network) cre automatiquement un composant rseau (CustomTCPServerContainer ou CustomTCPThreadedClient), si lapplication est de type serveur (AppManager.APP_SERVER) ou respectivement client (AppManager.APP_CLIENT).
